Time and Space Complexity

Time complexity tells us how efficient an algorithm is when solving a problem as the input size grows.

Space complexity means how much memory (RAM) an algorithm needs as the input size grows.

    O(N) means the work (space) grows in direct proportion to the input size.
    Example:
Suppose you have N students and you want to store their names in a list.
    For 10 students → you need space for 10 names.
    For 100 students → you need space for 100 names.
    For N students → you need space for N names.
    So the memory grows in linear proportion → that’s O(N) space
