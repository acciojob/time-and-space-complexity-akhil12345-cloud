Time and Space Complexity

Time complexity means how the time taken by a program grows when the input size grows. The efficient of time complexity means how well an algorithm handles bigger inputs.

Space complexity means how much memory (RAM) an algorithm needs as the input size grows.

    O(1) → Constant space → Algorithm uses the same memory, no matter the input size. Example: Swapping two numbers, simple math operations.
    O(n). → Linear space → Needs memory proportional to input. Example: Storing an array of N elements.
    O(n²) → Quadratic space → Memory grows very fast. Example: Storing an NxN matrix.
